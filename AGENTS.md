# Repository Guidelines

## 프로젝트 목표

- 쌍칼 웹 스튜디오의 반응형 비즈니스 홈페이지다.
- HTML, CSS, JavaScript만 사용하는 정적 웹사이트다.
- 별도 빌드 없이 GitHub Pages에서 정상적으로 동작해야 한다.

## 프로젝트 구조

- `index.html`: 문서 구조, 콘텐츠, 메타데이터
- `style.css`: 디자인, 테마, 반응형 레이아웃
- `script.js`: 테마 전환, 모바일 메뉴, 동적 기능
- `README.md`: 실행 및 배포 안내

상대 경로를 사용하고, 필요한 경우 정적 자산은 `assets/` 아래에 분류한다.

## 반드시 유지할 기능

- 라이트 모드와 다크 모드
- `localStorage`를 이용한 테마 저장
- 모바일 메뉴와 반응형 레이아웃
- 키보드 접근성 및 올바른 포커스 처리
- 본문 바로가기 링크

기능을 변경할 때는 위 항목이 퇴행하지 않는지 반드시 확인한다.

## 작업 원칙

- 작업 전에 변경 계획을 먼저 설명한다.
- 기존 코드의 좋은 구조와 동작은 유지한다.
- 요청에 필요한 파일과 부분만 최소한으로 수정한다.
- 승인 없이 외부 라이브러리나 패키지를 추가하지 않는다.
- API 키, 비밀번호, 토큰 등 비밀정보를 코드에 넣지 않는다.
- 실제 이메일, 전화번호, 주소를 임의로 만들지 않는다.
- 가짜 고객 후기, 매출, 성과를 만들지 않는다.
- 샘플 콘텐츠나 작업물은 반드시 샘플이라고 명시한다.
- 시맨틱 HTML을 사용하고 `aria-*`, 레이블, 키보드 조작 등 접근성을 지킨다.
- 현재 저장소 밖의 파일은 수정하지 않는다.

## 실행 및 검증

로컬 서버는 저장소 루트에서 다음과 같이 실행한다.

```bash
python -m http.server 8000
```

작업 후 데스크톱과 360px, 390px, 430px 모바일 화면을 확인한다. 라이트·다크 모드, 테마 저장, 모바일 메뉴, 키보드 탐색, 본문 바로가기 링크, 콘솔 오류도 검증한다. 완료 보고에는 변경 파일, 테스트 결과, 남은 문제를 포함한다.
