# 쌍칼 웹 스튜디오 홈페이지

비즈니스 홈페이지 제작 서비스를 소개하는 반응형 단일 페이지 웹사이트입니다. 별도의 빌드 과정이나 외부 라이브러리 없이 HTML, CSS, JavaScript만으로 구성되어 있으며 라이트/다크 모드와 모바일 메뉴를 지원합니다.

공개 홈페이지: https://ssangkalweb.github.io/ssangkal-web-studio/

## 주요 기능

- 데스크톱, 태블릿, 모바일 반응형 레이아웃
- 사용자의 시스템 설정을 반영하는 라이트/다크 모드
- 선택한 테마를 브라우저에 저장
- 키보드 접근성을 고려한 모바일 메뉴와 본문 바로가기 링크
- 업종별 정보 구조와 문의 동선을 보여주는 작업 사례 섹션
- GitHub Pages에서 동작하는 상대경로 및 페이지 내부 링크
- 검색엔진 기본 메타데이터(title, description, robots, canonical)
- Open Graph 및 Twitter 공유 메타데이터
- 저장소 내부 SVG 파비콘
- 단일 공개 주소를 안내하는 `sitemap.xml`
- 홈페이지 복귀 링크를 제공하는 맞춤형 404 페이지

작업 사례 섹션의 모든 콘텐츠는 실제 고객 작업이 아닌 **연습용 샘플**입니다.

## 로컬 실행 방법

빌드나 패키지 설치는 필요하지 않습니다.

1. 저장소를 내려받거나 복제합니다.
2. 프로젝트 폴더의 `index.html`을 브라우저에서 엽니다.

로컬 웹 서버로 확인하려면 프로젝트 폴더에서 다음 명령을 실행한 뒤 `http://localhost:8000`에 접속할 수 있습니다.

```bash
python -m http.server 8000
```

## GitHub Pages 배포

GitHub 저장소의 **Settings → Pages**에서 배포할 브랜치와 루트 폴더를 선택하면 됩니다. 사이트 파일은 저장소 루트에 있으며 별도의 빌드 설정은 필요하지 않습니다.

## 파일 구성

- `index.html`: 페이지 구조, 콘텐츠, 검색·SNS 공유 메타데이터
- `style.css`: 반응형 디자인 및 다크 모드 스타일
- `script.js`: 테마 전환, 모바일 메뉴, 연도 표시 기능
- `favicon.svg`: 브라우저 탭에 표시되는 SK SVG 아이콘
- `sitemap.xml`: 검색엔진에 공개 홈페이지의 단일 URL을 알리는 사이트맵
- `404.html`: 잘못되거나 없는 경로에서 안내와 홈페이지 복귀 버튼을 제공하는 독립형 페이지

## 운영 준비 항목 확인

- 홈페이지에서 브라우저 탭 제목과 파비콘을 확인합니다.
- 페이지 소스의 `<head>`에서 description, robots, canonical, Open Graph, Twitter 메타데이터가 각각 한 번만 선언되었는지 확인합니다.
- `https://ssangkalweb.github.io/ssangkal-web-studio/sitemap.xml`에서 공개 주소 하나만 표시되는지 확인합니다.
- 존재하지 않는 경로로 접속해 맞춤형 404 페이지와 홈페이지 복귀 버튼을 확인합니다.
- SNS 공유 미리보기 도구를 사용할 때 실제 공유 이미지가 아직 없으므로 이미지 없이 제목과 설명만 표시되는 것이 정상입니다.

## 문의 방식

실제 공개용 문의 이메일 연결을 완료했습니다. 문의 버튼은 사용자의 기본 메일 프로그램 또는 등록된 메일 처리기를 여는 `mailto:` 방식으로 작동하며, 별도 서버나 폼 데이터 저장 기능은 사용하지 않습니다.
